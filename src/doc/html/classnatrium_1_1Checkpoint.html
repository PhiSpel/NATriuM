void write (const Mesh< dim > &mesh, const DistributionFunctions &f, const dealii::DoFHandler< dim > &dof_handler, const CheckpointStatus &status)
 Write checkpoint to checkpoint directory. A checkpoint contains of three files:

  1. a .stat file that contains some information about the status of the simulations (iteration number, time, stencil scaling, ...).
  2. a .data file that contains the serialized data (mesh and distribution functions)
  3. a .data.info file that contains some deal.II information about the mesh, refinement etc.

void load (DistributionFunctions &f, ProblemDescription< dim > &problem, AdvectionOperator< dim > &advection, CheckpointStatus &status)
 Load checkpoint from file. A simulation can be resumed from a checkpoint if

  1. the mesh is exactly the same as in the previous simulation
  2. the mesh is a globally refined version of the previous simulation Restarts are also possible with arbitrary Mach number. Varying the order of finite elements is not supported, so far.
